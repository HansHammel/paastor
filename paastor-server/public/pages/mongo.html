MongoDB on Paastor

Paastor now supports simple MongoDB deployments. With a few clicks, you can have MongoDB running on any of your servers.

From the Server Manager, hit the MongoDB checkbox for one of your servers and fill out the required fields. Take note of any security information you set - it will not be saved by Paastor. In a few moments, it will be installed and you will be ready to connect.

Remote access to MongoDB

Often people use MongoDB in a trusted environment and do not allow it to receive remote connections.

If you opt for remote access, be aware of how to connect, and that it may be slightly different than connecting to local or cloud based MongoDB services. This is because local and cloud systems shield you from some of the normal MongoDB administration, and create granular permissions at the database level.

Paastor creates an admin user which you can use to connect to any database, and create new users for those databases.

However, you must specify that you are connecting as an admin user. This is because MongoDB stores database permissions inside each database.

You must tell MongoDB that your authSource is admin.

Querystring method
var mongoose = require('mongoose');
 
var mongoUri = "mongodb://username:password@3.3.3.3:27017/somedatabase?authSource=admin";
mongoose.connect(mongoUri);
Options object method
var mongoose = require('mongoose');
 
var mongoUri = "mongodb://username:password@3.3.3.3:27017/somedatabase";
var options = {
      auth: { authSource: 'admin' }
};
mongoose.connect(mongoUri, options);

Creating a MongoDB database

New databases are created immediately when you connect to them. There is no need to create one.
